# Vim × ThinkPadで人生が変わる話

キーボードから手を離さない世界へ

---

# こんなことしていませんか？

- マウスでカーソル移動
- クリックして位置を選択
- キーボードに戻って入力
- またマウスへ…

👉 この往復、無駄です

---

# 結論

## Vim × ThinkPad = 最強

- 手をホームポジションから動かさない
- 思考が途切れない
- 作業効率が上がる

---

# Vimの強み

- キーボードだけで完結
- カーソル移動もすべてキー操作

基本：
- `h j k l` → 移動
- `i / a` → 入力
- `ESC / jj` → ノーマルモード

👉 マウス不要

---

# ThinkPadの強み

- トラックポイントでポインタ操作
- ホームポジション維持

👉 手を動かさない設計

---

# 組み合わせるとどうなるか

## 完全キーボード作業環境

- Vim → テキスト操作
- トラックポイント → ポインタ操作

👉 思考が止まらない

👉 かなり快適

---

# 実はここまでできる

- PowerPointも普通に操作できる
- 図の作成もトラックポイントで対応可能

👉 「マウスがないと無理」は思い込み

---

# モバイル環境で強い

- 出先でもノートPCだけで完結
- マウス不要

👉 いつでも同じ作業環境

---

# 地味に重要なポイント

## かっこいい

- キーボードだけで操作
- 無駄な動きがない

👉 モチベーションが上がる

---

# 最初は地獄です

- Vim → 操作が難しい
- トラックポイント → 慣れない

👉 正直かなりイライラします

---

# でも1週間耐えると

- マウスに手を伸ばすのがダルくなる
- キーボードから離れたくなくなる

👉 明確に世界が変わる

---

# Vimの真価（ターミナル）

- bashと組み合わせて作業できる
- CLI中心のワークフローに統合可能

例：
- grep / ripgrep
- git操作
- ビルド・実行

👉 汎用性が一気に広がる

---

# 開発環境に縛られない

- Vim非対応の環境でもOK
- ターミナルで実体ファイルを開いて編集

👉 どこでも同じ操作感

---

# AIとも共存できる

- GitHub Copilotも利用可能
- Vim環境でも補完・生成が使える

👉 モダン開発とも両立
